## Reporting Bugs

### Before Reporting

1. Search existing issues — your bug may already be reported.
2. Check the documentation — the answer may be in the docs.
3. Reproduce the issue — ensure you can reproduce it consistently.
4. Update to the latest version — the bug may already be fixed.

### How to Report

1. Create an issue using the Bug Report template.
2. Include:
   - Clear description
   - Steps to reproduce
   - Expected vs actual behavior
   - Environment details (OS, Python version, runtime/backend)
   - Screenshots or logs (if applicable)

### After Reporting

- We will acknowledge your report within 48 hours.
- We may ask for clarification if needed.
- We will keep you updated on progress.

## Requesting Features

### Before Requesting

1. Check existing issues — your feature may already be requested.
2. Consider alternatives — is there another way to achieve your goal?

### How to Request

1. Create an issue using the Feature Request template.
2. Include:
   - Clear description
   - Use case
   - Alternatives considered
   - Additional context

## Frequently Asked Questions

**Q: What is ModelDock?**
A: A lightweight, Python-first model manager for local LLMs — the package manager for local AI models.

**Q: Is ModelDock free?**
A: Yes. ModelDock is open source under the Apache 2.0 license.

**Q: Which runtimes are supported?**
A: Ollama ships first. LM Studio, llama.cpp, Jan AI, GPT4All, and vLLM are planned as drop-in adapters.

**Q: How do I get started?**
A: See the Quick Start section in [README.md](README.md#quick-start) or [QUICKSTART.md](QUICKSTART.md).

**Q: How can I contribute?**
A: See [CONTRIBUTING.md](CONTRIBUTING.md).
